# 产品需求文档 (PRD) - AE AI Agent

**版本**：v2.1（MCP + 原生聊天 App 双引擎版）
**日期**：2026年3月30日
**状态**：Phase 4 规划中

---

## 1. 产品定位

### 1.1 产品愿景

构建一个面向 After Effects 场景的原生 AI 工作台，形成“双引擎”产品形态：
- **开放引擎**：通过 MCP Server 向任意 LLM 客户端开放 AE 工具能力
- **自有引擎**：通过原生 macOS 聊天 App 提供深度优化的 AE 智能体验（记忆、经验、知识、多智能体）

### 1.2 Phase 4 目标

在不破坏现有 MCP/CEP 架构的前提下，实现以下结果：
1. 发布可运行的 macOS 原生聊天 App（MVP）
2. 建立“记忆层 + 经验层 + 知识层”中间系统，而非单轮聊天
3. 支持多智能体协作完成 AE 复杂任务
4. 形成可复用的工程化开发规范，供 AI 编程团队执行

### 1.3 成功指标（MVP）

| 指标 | 目标 |
|------|------|
| 首次连接成功率 | ≥ 95%（App 启动后可连接 AE MCP） |
| 任务成功率 | ≥ 85%（20个标准 AE 任务集） |
| 平均修复轮次 | ≤ 2 次（失败后自动纠错重试） |
| 首次可见结果时间 | ≤ 8 秒（用户输入到首次可观察动作） |
| 经验复用命中率 | ≥ 40%（相似任务触发经验模板） |

## 2. 市场策略与二开选型

### 2.1 市面实现共性（调研结论）

主流 AI Agent 产品普遍采用“聊天 UI + agent runtime + memory + tool orchestration”分层模式，关键不是单模型能力，而是中间层工程化能力：
- 显式规划与执行解耦（Planner/Executor/Verifier）
- 工具调用协议化（MCP / function calling）
- 持久化状态（thread/workspace/checkpoint）
- 运行时可观测（日志、重试、回放）

### 2.2 候选方案对比（最小成本导向）

| 方案 | 成本 | 可控性 | MCP 兼容 | 风险 | 结论 |
|------|------|--------|---------|------|------|
| **Cherry Studio 二开** | 低 | 中 | 高 | AGPL 合规成本 | **推荐用于 MVP 快速落地** |
| **Dive 二开** | 低-中 | 中 | 高 | macOS Tauri 路线仍在完善 | 作为备选 |
| **从零开发 Tauri App** | 高 | 高 | 高 | 工期和验证成本高 | 不建议作为 Phase 4 起步 |
| **LibreChat 私有部署后桌面封装** | 中 | 高 | 高 | 偏 Web 架构，桌面原生体验弱 | 适合企业私有化，不是首选 MVP |

### 2.3 技术选型决策

Phase 4 建议采用：
1. **桌面宿主**：Cherry Studio 二次开发（优先）
2. **Agent Runtime**：LangGraph（状态机 + 持久化 checkpointer）
3. **工作流自动化**：n8n（异步任务、外部集成、经验处理流水线）
4. **现有能力复用**：直接复用 `ae-mcp-server` 与 `ae-cep-extension`

## 3. 目标用户与核心场景

| 用户类型 | 关键痛点 | App 核心价值 |
|---------|---------|-------------|
| 初中级动效师 | 不懂表达式和 AE 术语 | 直接说目标，系统自动拆解并执行 |
| 高级动效师/TA | 重复操作多、模板迁移重 | 经验模板复用、批处理自动化 |
| 团队负责人 | 产出标准不一致 | 共享经验库与可审计执行日志 |

### 3.1 高价值场景

1. **一步到位场景**：自然语言 → 自动完成图层/关键帧/效果设置
2. **模板迁移场景**：读取参考合成 → 克隆结构 → 批量替换内容
3. **教程固化场景**：输入 YouTube 教程链接 → 提炼为经验模板并可复用
4. **失败自修场景**：执行失败后自动修复参数并重试

## 4. 产品功能需求（原生 App）

### 4.1 信息架构

1. **会话区**：主聊天流、步骤状态、工具调用卡片
2. **任务区**：计划树、执行进度、失败重试状态
3. **记忆区**：当前项目记忆、历史经验命中、知识引用
4. **工具区**：MCP 工具开关、风险标记、执行权限
5. **设置区**：模型路由、AE 连接、n8n 集成、数据策略

### 4.2 P0 功能（MVP 必做）

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| AE 连接管理 | 检测/重连 MCP Server，展示实时状态 | 启动后 3 秒内给出连接结果 |
| Agent Loop 执行器 | analyze → plan → act → verify 循环 | 每轮执行都有可视状态 |
| 工具检索与执行 | 先 `search_ae_tools`，再调用具体工具 | 未知工具参数时强制先检索 |
| 失败自纠错 | 错误自动重试（最多3次） | 失败原因可见、重试轨迹可回放 |
| 任务工作区 | 每任务独立 workspace 和运行日志 | 可查看本任务中间产物 |
| 会话持久化 | thread 级会话恢复 | 重启 App 后可继续任务 |

### 4.3 P1 功能（MVP 后）

| 功能 | 描述 |
|------|------|
| 经验库命中推荐 | 根据用户意图推荐历史成功策略 |
| YouTube 教程固化 | 视频转步骤卡与可执行 playbook |
| 多智能体协同面板 | 展示 Planner / Executor / Verifier 分工 |
| 任务模板市场 | 团队共享可复用经验模板 |

## 5. 记忆、知识、经验三层系统

### 5.1 记忆层（Memory）

- **短期记忆**：当前线程上下文、执行状态、最近观察
- **长期记忆**：任务结果、错误案例、用户偏好（存储到持久化数据库）
- **机制**：基于 thread_id 持久化，支持任务中断恢复

### 5.2 知识层（Knowledge）

- **来源**：AE 官方文档、内部 SOP、项目规范
- **能力**：检索增强（RAG）+ 引用可追踪
- **策略**：高频知识预索引，长文按需切片召回

### 5.3 经验层（Experience）

- **核心对象**：Playbook（触发条件 + 步骤序列 + 常见错误修复）
- **来源**：
  1. 人工沉淀（专家规则）
  2. 成功任务自动抽取
  3. 教程内容结构化转译
- **执行方式**：优先命中经验模板，再做增量规划

## 6. 多智能体设计

| 智能体 | 职责 | 输入 | 输出 |
|-------|------|------|------|
| Planner | 理解意图并生成计划 | 用户目标 + 上下文 | 结构化 plan |
| Executor | 调用工具执行步骤 | plan + 工具 schema | 观察结果 |
| Verifier | 校验目标达成与质量 | 结果 + 验收标准 | 评分 + 修复建议 |
| Memory Curator | 管理记忆与经验沉淀 | 执行日志 + 对话 | 经验候选条目 |
| Knowledge Retriever | 拉取文档证据 | 查询意图 | 证据片段 |

## 7. n8n 集成策略

### 7.1 集成边界

将 n8n 用于“异步与周边流程”，不进入 AE 核心控制链路：
- 教程抓取与转写流水线
- 经验条目审核流程
- 通知、报表、归档任务
- 定时训练与评估任务编排

### 7.2 不纳入 n8n 的部分

- 实时 AE 工具调用
- 高时序依赖的 Agent Loop 决策
- 需要毫秒级状态一致性的执行路径

## 10. 验收标准（Phase 4 完成定义）

1. 在 macOS 上可独立运行原生聊天 App
2. 能通过现有 MCP 工具完成 20 个标准 AE 任务，成功率≥85%
3. 可展示任务计划、执行、验证、重试全流程
4. 会话与任务状态重启后可恢复
5. 至少支持 10 条经验模板命中并成功复用
6. 文档、架构、实现与 `specs/API.md`、`docs/ARCHITECTURE.md` 一致

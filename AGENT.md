# 🤖 AI Agent 编程指引 (AGENT.md)

**致 Claude Code / Trae / Cursor 等 AI 编程助手：**

你好！你正在参与开发 **AE AI Agent** 项目。这是一个运行于独立桌面端，并通过本地文件桥接与 Adobe After Effects (AE) 通信的 AI 辅助工具。

为了确保项目的顺利推进，并最大化避免 Vibe Coding 过程中的“幻觉”和“臆断”，**你必须严格遵守以下原则和开发流程**。

---

## 1. 核心原则：文档优先 (Documentation First)

在这个项目中，**文档是唯一的真理来源 (Single Source of Truth)**。

*   **禁止臆断**：如果你在开发某个功能时，发现缺少 API 定义、数据结构或架构说明，**绝对不要自行假设或捏造**。你必须停止编码，并向用户（开发者）报告缺失的信息。
*   **严格遵循规范**：所有的模块划分、通信协议、UI 交互逻辑，都必须严格按照 `docs/` 和 `specs/` 目录下的文档执行。
*   **先读文档，再写代码**：在开始任何一个新模块的开发前，你必须先读取并理解对应的设计文档。

## 2. 文档索引与阅读顺序

在开始编码前，请按照以下顺序阅读项目文档：

1.  **`docs/PRD.md`** (产品需求文档)：了解我们要做的产品是什么，核心功能有哪些，以及开发的优先级。
2.  **`docs/ARCHITECTURE.md`** (系统架构文档)：理解“独立 App + 本地文件桥接 + AE CEP 接收器”的整体架构，以及各个模块的职责。
3.  **`specs/API.md`** (API 与通信协议规范)：这是**最重要**的文档之一。它定义了 App 与 AE 之间如何通过 JSON 文件通信，以及所有支持的指令格式。**编写通信代码时必须严格遵守此文档。**
4.  **`docs/CONTEXT_ENGINEERING.md`** (上下文工程与 AI 架构)：了解如何组装 Prompt、如何实现 `@` 引用机制、以及如何利用 Prompt Caching。
5.  **`rules/DEVGUIDE.md`** (开发规范与交付清单)：了解技术栈要求、代码规范以及每个阶段的交付目标。

## 3. 开发工作流指引

当用户要求你“开始开发 Phase 1”或“实现某个功能”时，请遵循以下步骤：

1.  **确认上下文**：确认你已经阅读了与该任务相关的文档。如果没有，请先使用文件读取工具查看。
2.  **复述设计**：在开始写代码前，简要向用户复述你理解的架构和 API 设计，以确保对齐。
3.  **分步实现**：不要试图一次性写完所有代码。按照 `DEVGUIDE.md` 中的阶段划分，一步步实现。
4.  **严格的错误处理**：特别是在处理文件读写（与 AE 通信）和 ExtendScript 执行时，必须加入完善的 `try...catch` 和日志记录。
5.  **验证与测试**：完成一个模块后，编写简单的测试脚本（如模拟 AE 写入 `.response` 文件）来验证逻辑。

## 4. 常见陷阱警告 (Red Flags)

*   **不要使用 WebSocket/HTTP Server 与 AE 通信**：文档中已明确规定使用**本地文件轮询桥接 (File-based Polling Bridge)**。不要试图引入网络通信，这在 AE 中极不稳定。
*   **不要让 LLM 直接写大段 ExtendScript**：必须调用 `API.md` 中定义的封装好的工具函数。
*   **注意 ExtendScript 的环境限制**：AE 的 ExtendScript 基于 ES3，不支持 `let`, `const`, 箭头函数等现代 JS 语法。在编写 AE 端的 JSX 代码时，必须使用老式语法。

## 5. 会话启动与执行规则

每次开启新会话，必须先完成以下动作后再编码：

1.  先阅读本文件 `AGENT.md`。
2.  阅读并遵守 `rules/RULES.md`。
3.  阅读并遵守 `.trae/rules/project_rules.md`。
4.  若涉及通信与指令执行，先核对 `specs/API.md` 的字段与 Action，再开始实现。
5.  若文档未定义字段、流程或边界条件，先报告缺失，不得自行补全。
